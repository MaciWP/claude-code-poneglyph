05 Lo Que Hacemos Bien

1. Patron Lead Orchestrator

▼

Delegacion pura con hooks de enforcement. El Lead nunca ejecuta codigo directamente -- delega a builder, reviewer, planner, scout y error-analyzer. Esto es el modelo de orquestacion mas sofisticado de los tres sistemas.

PI Agent permite un patron similar via extensiones, pero requiere construccion manual. Stripe tiene orquestacion custom pero no documentan un patron de delegacion pura como el nuestro.

2. Ecosistema de Agentes

▼

17 agentes especializados es el catalogo mas grande de los tres sistemas. Incluye: builder, reviewer, planner, scout, error-analyzer, architect, code-quality, security-auditor, refactor-agent, task-decomposer, merge-resolver, bug-documenter, knowledge-sync, test-watcher, command-loader, browser-qa y extension-architect.

Cada agente tiene un rol bien definido con herramientas permitidas/prohibidas, lo que fuerza la especializacion.

3. Auto-Loading de Skills por Keywords

▼

24 skills con matching por keywords y prioridad. El sistema detecta automaticamente que skills necesita un agente basandose en el prompt del usuario. Unico entre los tres sistemas.

PI Agent soporta skills pero requiere seleccion manual. Stripe usa MCP tools pero no documenta auto-matching.

4. Puertas de Calidad de Prompt

▼

Scoring de 5 criterios (Clarity, Context, Structure, Success, Actionable) + routing por complejidad de 5 factores. Esto previene desperdicio de tokens en prompts vagos y enruta tareas al nivel correcto de planificacion.

Ningun otro sistema tiene esto documentado.

5. Memoria Semantica

▼

Vector store + graph store con hooks de inyeccion. Ni PI ni Stripe documentan capacidades de memoria semantica. Esto nos da una ventaja unica en retener contexto entre sesiones y aprender de ejecuciones pasadas.

6. Learning Loop

▼

Auto-mejora desde trazas de ejecucion. Completamente unico. El sistema aprende de patrones de exito y fracaso, ajustando comportamiento futuro automaticamente.

7. Web UI con Streaming

▼

React UI completa con WebSocket streaming, dashboard de metricas y panel de QA. Supera la terminal de PI. Comparable a la UI interna de Stripe.

8. Sistema QA Integrado

▼

Playwright con YAML stories integrado en el dashboard. Permite testing visual automatizado como parte del flujo de desarrollo agente.

9. Pipeline de Validacion

▼

Hooks PostToolUse (formato, typecheck) + hooks Stop (tests) crean guardrails de calidad. Cada edicion se valida automaticamente, cada parada verifica que los tests pasen.

10. Multi-Proveedor

▼

Claude + Codex + Gemini da flexibilidad para elegir el mejor modelo segun la tarea. PI gana aqui con soporte para cualquier modelo, pero nosotros cubrimos los tres principales.

06 Brechas a Cerrar

Alta prioridad

1. Agent Sandbox / DevBox

Necesitamos entornos de ejecucion aislados. Actualmente todos los agentes comparten el filesystem. El patron DevBox de Stripe demuestra el valor: instancias EC2 como sandboxes con 10s de warm startup.

Solucion propuesta: Contenedores Docker o VMs cloud. Pre-warmed pool con inicializacion de git worktree.

Alta prioridad

2. Blueprint Engine / Mejora de Workflows

Nuestro WorkflowExecutor es un buen inicio pero necesita integracion mas profunda. El interleaving de codigo + agentes de Stripe es el gold standard. Necesitamos: pasos deterministicos inline (linting, git, testing) entre pasos de agente.

Solucion propuesta: Mejorar WorkflowExecutor con nodos deterministicos, formato YAML de Blueprint, transiciones basadas en eventos.

Media prioridad

3. Agent Chains / Pipelines

Formalizar la ejecucion secuencial de agentes mas alla de WorkflowExecutor. El concepto de cadena de PI (agente → agente → agente) es simple pero poderoso.

Media prioridad

4. Modo Out-of-Loop (Outloop)

Crear ejecucion totalmente desatendida disparada via API/Slack/cron. El Lead Orchestrator es interactivo hoy; deberia soportar tambien modo fire-and-forget.

Media prioridad

5. Carga de Contexto Condicional

Las reglas glob-based de Stripe son inteligentes. Cargar las 10 reglas incondicionalmente desperdicia contexto. Implementar activacion de reglas consciente del directorio.

Media prioridad

6. Mas Puntos de Hook

PI tiene 25+ eventos de hook. Nosotros tenemos 6. Considerar agregar: OnAgentStart, OnAgentEnd, OnWorkflowStep, OnError, OnContextCompact.

Baja prioridad

7. Extension Stacking

Las extensiones composables de PI son elegantes pero podemos lograr resultados similares con nuestro stacking de skills.

Baja prioridad

8. Tool Override

Nice to have pero el codigo cerrado de Claude Code limita esto. No es critico.

Baja prioridad

9. Integracion PI

Considerar agregar PI como harness alternativo de agentes (estrategia 80/20 del video).

07 Hoja de Ruta Estrategica

Fase 1: Blueprint Engine v2

2-3 semanas
  • Mejorar WorkflowExecutor para soportar pasos deterministicos inline
  • Agregar linter, git commit, ejecucion de tests como nodos deterministicos
  • Formato YAML de Blueprint para definir workflows
  • Transiciones de pasos basadas en eventos

Fase 2: Modo Outloop

2-3 semanas
  • Ejecucion desatendida via API (POST /api/execute-outloop)
  • Integracion Slack para disparar y revisar
  • Creacion de GitHub PR como estado final
  • Iteraciones maximas configurables / presupuesto de tokens

Fase 3: Agent Sandbox

3-4 semanas
  • Contenedor Docker por ejecucion de agente
  • Pool de contenedores pre-warmed
  • Aislamiento de filesystem con git worktree init
  • Extraccion de resultados y merge back

Fase 4: Orquestacion Avanzada

2-3 semanas
  • Agent chains/pipelines como concepto de primera clase
  • Carga de contexto condicional (reglas glob-based)
  • Mas puntos de hook (OnAgentStart, OnAgentEnd, etc.)
  • Seleccion dinamica de herramientas (concepto Tool Shed)

08 El Veredicto

Estamos en el buen camino? SI, absolutamente.

Nuestro sistema Poneglyph ya es mas sofisticado que el 95% de lo que tienen los ingenieros. Estamos por delante de PI en sofisticacion de agentes out-of-box y por detras de PI en personalizacion del harness. Nos acercamos a la arquitectura de Stripe pero necesitamos su profundidad en Blueprint Engine y aislamiento DevBox.

La Estrategia 80/20 tambien tiene sentido para nosotros:

80% Poneglyph (nuestro Claude Code personalizado) para trabajo diario

20% experimental con PI o harness custom para casos edge

Insight clave de Stripe:

"La especializacion es la ventaja" -- y ya estamos profundamente especializados con 17 agentes, 24 skills y 10 reglas. La siguiente frontera es Blueprint Engine v2 (interleaving deterministico + agente) y automatizacion Outloop.

Escalera de Madurez

1 Agente base (Claude Code out-of-box) La mayoria de ingenieros
2 Mejor agente (CLAUDE.md + skills)
3 Context engineering (rules + skills + hooks)
4-5 Agentes personalizados + Orquestador (Poneglyph) ESTAMOS AQUI
6 Meta-agentes (agentes construyendo agentes) Exploracion iniciada
7 ZTE (Zero Touch Engineering) NORTH STAR

Estamos solidamente en el Nivel 4-5. El siguiente paso es profundizar el Nivel 5 (Blueprint Engine) y comenzar la exploracion del Nivel 6 (Meta-agentes).
