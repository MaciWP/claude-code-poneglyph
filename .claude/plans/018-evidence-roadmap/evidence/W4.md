## Angle (a) — Dev-vs-deployed patterns in config managers (pattern analysis)

**Headline: chezmoi's copy-on-explicit-apply is the only surveyed model that kills the deploy-on-checkout hazard AND works first-class on Windows.**

- **chezmoi** (pattern[doc], all quotes refuter-verified, https://www.chezmoi.io): three-state model — source state (repo) → computed target state → destination. The destination changes only on explicit `chezmoi apply` (supported via add/apply semantics + status wording; note: "inert" is our characterization, not a docs quote — refuter precision). Design FAQ verbatim: chezmoi "generates the dotfile as a regular file in its final location" to enable templates/encryption/permissions, and **admits the trade-off**: "The only advantage to using GNU Stow-style symlinks is that changes… are immediately visible… whereas chezmoi currently requires you to pass the `--watch` flag". Drift detection: `chezmoi status` column 1 = last-written vs actual (manual edits), column 2 = "what effect running `chezmoi apply` will have" (repo ahead of machine). `chezmoi update` = pull --autostash --rebase + apply. Per-machine: config-file variables + hostname/OS template conditionals + templated `.chezmoiignore` — maps 1:1 onto poneglyph's `settings.json` + `settings.machine.json` overlay.
- **nix home-manager** (pattern[doc]): build-to-store → atomic activate (`switch`), generations with `switch --rollback`, flake.lock pinning. Gold standard for atomicity; toolchain cost disproportionate for this case and **no native Windows**.
- **GNU stow** (pattern[doc], man page): "symlink farm manager… creating symlinks in the target tree that point into the package tree" — **the same flaw as poneglyph's current sync**: any checkout in the repo is instantly live; no release layer. **dotbot**: explicit `./install` but symlinks by default → apply step governs links, not content. **Bare-git-repo**: $HOME IS the work tree — the flaw embraced deliberately, managed by one-branch-per-machine discipline (Tier B, search-corroborated).
- **Staleness signals**: `chezmoi status/diff/verify` (verify exits non-zero on mismatch — usable in prompts/CI); home-manager generation list. **No published off-the-shelf "version-stamp + prompt warning" pattern exists** (UNVERIFIED as canon) — it's a small custom build modeled on chezmoi's status column 2.
- **Counter-evidence**: copy-model friction is documented — forget-to-apply loops (Tier B/C user reports), external-edit drift until `status`/`re-add` (chezmoi#1348); the instant-feedback advantage of symlinks is real and admitted by chezmoi's own FAQ (mitigated by watch mode, not denied).

| Property | Symlink-to-worktree (current) | Copy-on-apply (chezmoi model) | Generations (home-manager) |
|---|---|---|---|
| `git checkout` deploys silently | **Yes** | No | No |
| Dev feedback | instant | needs apply/`--watch` | slowest |
| Drift detection | none | 3-state status | generation diff |
| Windows | junction pain | first-class | poor/none |
| Failure mode | branch state deployed everywhere | forgetting to apply | heavy toolchain |

## Angle (b) — Memory architectures with published evals

**Headline: plain files + full-context beat every specialized memory product; selective curation beats add-all 3×; sync is unsolved officially — but the official `autoMemoryDirectory` lever exists.**

- **Full-context beats all memory systems — in Mem0's OWN paper** (Tier A/B, Table 2, refuter-verified, arXiv 2504.19413): full-context J **72.90** > Mem0-graph 68.44 > Mem0 66.88 > Zep 65.99 > LangMem 58.10. Memory's measured win is cost/latency (p95 1.44s vs 17.1s; >90% token saving) — not quality. (Abstract's "+26%" is vs OpenAI memory, not vs full-context — cite the table.)
- **Benchmark war = noise**: Zep 84% → Mem0-replicated 58.44% → Zep counter 75.14% (dual-COI dispute); 2026 vendor self-reports cluster 83-92.5% and are mutually incomparable. Rankings within ~10pts are not evidence.
- **Letta**: plain **filesystem tools (grep/search/open) 74.0% vs Mem0-graph 68.5%** on LoCoMo (Tier B, Letta COI) — consistent with W3's files-beat-graph finding. **Claude Code's native markdown-files memory is already the evidence-supported architecture.**
- **Selective beats add-all 3×** (Tier A — primary located by refuter: arXiv 2505.16067, "experience-following behavior", May 2025): EHRAgent medical reasoning **38.86% selective vs 13.04% add-all** (store grown to 2,411 records); mechanism: agents reproduce the quality of whatever they retrieve, no uncertainty signal on stale entries. **Memory poisoning is demonstrated and persistent** (Tier A: arXiv 2512.16962, 2601.05504) — applies to any auto-written store incl. MEMORY.md.
- **Claude Code official surface** (Tier A, refuter-verified, https://code.claude.com/docs/en/memory): auto-memory is per-git-repo and **machine-local** ("not shared across machines"); MEMORY.md loads first 200 lines / 25KB per session; **`autoMemoryDirectory` setting redirects storage** — the official lever for a git-synced vault. No cross-project or cross-machine mechanism exists or is announced (open FR #25739).
- **Cross-project promotion**: the only patterns with published transfer numbers are Voyager skill libraries and AWM workflow induction (+24.6%/+51.1% rel, cross-domain) — both promote **procedural artifacts** (skills/rules), not raw episodes. No published "constitution distillation" pipeline with evals (UNVERIFIED gap) — poneglyph's retro→promotion loop matches the evidenced pattern.
- **Sync patterns** (all Tier C, none evaluated): git-synced vault; daemon mirror of stock paths; age-encrypted per-machine snapshots (agentsync); Letta server model (overkill). Cleanest official fit: `autoMemoryDirectory` → git-synced vault repo.

### Claude Code sandbox, current state (Tier A, refuter-verified, https://code.claude.com/docs/en/sandboxing)

- OS-level: **macOS Seatbelt (nothing to install)**, Linux/WSL2 bubblewrap+socat; **"Native Windows is not supported"**. Enforced on process + children — holds regardless of what the model runs (vs permission rules that evaluate the command STRING — the bypassable layer).
- Defaults that matter: write = cwd + $TMPDIR; **read = entire disk, explicitly including `~/.aws/credentials` and `~/.ssh/` — docs say "Add them to `denyRead` to block them"**; network domains prompt on first use; proxy checks hostname **without TLS inspection** → broad domains (github.com) remain documented exfil paths.
- **"sandboxing safely reduces permission prompts by 84%"** (internal usage `[vendor]`) — auto-allow-in-sandbox arguably beats prompt-everything-unsandboxed (approval fatigue is Anthropic's own stated rationale).
- Scope gaps: Bash+children only (Read/Edit/Write go through permissions); env vars inherited (scrub via `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB`). Friction: watchman/jest, docker, Go-TLS CLIs (gh/gcloud/terraform) need `excludedCommands`.
- `--dangerously-skip-permissions`: official guidance = only inside containers/devcontainers.

### Measured risk

- **AgentDojo** (Tier A, NeurIPS 2024, refuter-verified): injection succeeds **<25%** of 629 cases vs best agents; tool-filter defense → **7.5%** ASR. InjecAgent: GPT-4 ReAct 24% base.
- MCP ecosystem scans (Tier B, methodology varies): 33-66% of scanned servers with findings; 43% command-injection in one firm's tests; academic OSS scan 5.5% tool-poisoning. Range, not point estimate.
- **No documented in-the-wild exploitation of a solo dev's Claude Code session via prompt injection was found** — realized losses are platform agents with prod access, supply-chain installs, and patched CVEs (absence-of-evidence, not evidence-of-absence; declared).

## Verdict

1. **Sync v2 direction**: copy-on-explicit-apply from a committed ref (main/tag), chezmoi-model — replicated inside `sync-claude` rather than adopting chezmoi wholesale; keeps the settings.machine.json overlay pattern (it IS chezmoi's per-machine pattern); add a dev-mode escape hatch for instant-feedback work inside poneglyph; staleness = stamp (SHA at apply) vs origin/main in statusline — small custom build, no published off-the-shelf pattern.
2. **Memory**: keep native markdown architecture (evidence says don't add Mem0/Zep/Letta); harvest = selective promotion of procedural artifacts via the existing retro loop (the only evidenced pattern); **curation is mandatory** (3× degradation from add-all; poisoning persists); sync via `autoMemoryDirectory` → git-synced vault is the official-surface option (Tier C patterns only — pilot, don't trust).
3. **Security posture options** (for the user to pick, 019+): (1) trust+backup declarado — zero friction, accepts credential-exfil exposure under injection; (2) **sandbox-lite — the evidence's recommendation on macOS**: `sandbox.enabled` + auto-allow + `denyRead` creds + env scrub ≈ free, FIXES the string-rule bypass class (CVE-2025-54794 was exactly that), and reduces prompts 84% `[vendor]`; Windows machine stays on posture 1 (no native sandbox); (3) containerize only untrusted repos + unattended runs.
4. Supply-chain hygiene is posture-independent: no marketplace skills, pin MCP versions, never open untrusted repos in auto mode (CVE-2025-59536 class), keep CC updated.
5. Honest gaps: no measured comparison of sync patterns; no in-the-wild solo-dev exploitation data; sandbox exfil guarantees are partial by Anthropic's own documentation.
