## Angle (a) — Best-of-N + mechanical verifier in shipped interactive tools

**Headline: best-of-N is table stakes in products (2025-26), but NO shipped tool selects by tests — selection is always human or LLM judge.**

- **Cursor 2.0** (Oct 2025): "Best of N" up to 8 agents, same prompt, worktree each, **human picks** (Tier B, https://cursor.com/blog/2-0). **Cursor 2.2** (Dec 2025): Multi-Agent Judging — LLM judge evaluates all runs after the slowest finishes, marks winner with explanation (Tier B, https://cursor.com/changelog/2-2). Cursor staff state the judge "doesn't specifically *optimize for* code size, style, architectural choices, or cost" (Tier B/C, staff post: https://forum.cursor.com/t/cursor-2-2-multi-agent-judging/145826 — refuter-corrected wording). Forum users report manually combining outputs beats the judge's pick (Tier C/D, same thread).
- **OpenAI Codex** (Jun 2025): best-of-n shipped — multiple simultaneous attempts, **human picks** aided by preview screenshots (Tier B, https://openai.com/index/introducing-upgrades-to-codex/); practitioner reports 2-4 candidates/task (Tier C, zackproser.com 2026 review).
- **Windsurf Arena Mode** (Wave 14, Jan 2026): 2 hidden-identity agents, **human votes**, Elo leaderboards; double run priced as single — a deliberate subsidy signal (Tier B, https://devin.ai/blog/windsurf-wave-14).
- **Devin**: parallel instances = different tasks (multitasking), NOT same-task best-of-N (Tier B, https://cognition.ai/blog/devin-2).
- **Claude Code**: no built-in best-of-N selection (absence verified in official docs, fetched 2026-06-10). DIY primitives exist: `claude -p --worktree` non-interactive fan-out; **worktrees are NOT auto-cleaned with `-p`** ("Remove them with `git worktree remove`") (Tier B, https://code.claude.com/docs/en/worktrees — refuter-confirmed verbatim).
- **OSS harnesses**: `uzi` (N agents → worktrees+tmux → human checkpoint-merge), `claude-squad`, `gwq` — none ships automated test-based selection (Tier C, github.com/devflowinc/uzi).
- **Only measured solo run found**: 4 agents (2 CC + 2 Codex), 25% useful-output rate per agent → ~68% chance ≥1 success; $0.40 total; ~20 min saved; human selection (Tier C, https://dev.to/skeptrune/llm-codegen-go-brrr-parallelization-with-git-worktrees-and-tmux-2gop).
- **Gap declared**: no published latency/cost/quality data for N=2-5 with TEST-based selection in real interactive workflows (absence after search, 2026-06-10). Running that loop generates novel evidence.

## Angle (b) — Background sessions: independent outcome data + self-measurement

**Headline: independent merged/ALL rates are 43-83% by agent (not the 96-98% merged/ready vendor-adjacent counters), and the #1 failure mode is reviewer abandonment.**

- **arXiv 2601.15195** ("Where Do AI Coding Agents Fail?", Jan 2026, AIDev-derived, 33,596 PRs, repos ≥100 stars, Tier A preprint — refuter-confirmed verbatim): merge rates Codex **82.59%**, Cursor 65.22%, Claude Code 59.04%, Devin 53.76%, Copilot 43.04%; overall **71.5%**. Each failed CI check ≈ **−15% merge odds**. Unmerged PRs are 17% larger, touch 10% more files. **38% of the studied rejected-PR sample (228 PRs) = reviewer abandonment** (refuter caveat: % of analyzed sample, not of all rejections); 23% duplicates; 17% CI failures.
- **Denominator trap**: ai-pr-watcher counters (Codex 87.76% as of 2026-06-10) compute merged/**ready** — drafts/abandoned excluded; AIDev-family studies compute merged/**all**. Not comparable; use merged/all for pilot math (Tier C, github.com/aavetis/ai-pr-watcher).
- **arXiv 2601.16809** ("Will It Survive?", **submitted to EASE 2026, under review** — refuter venue correction; 5,171 PRs, survival analysis): agent-authored lines die LESS than human (53.9% vs 69.3%; HR 0.842, 95% CI 0.833-0.852, p<0.001) but draw more **corrective** modifications (26.3% vs 23.0%, small effect, Cramér's V 0.116). Per-agent line-death: Cursor 38.7%, Claude Code 41.0%, **Devin 71.7% — worse than human baseline**. Net quality direction is genuinely contested.
- **~1/3 of merged agent PRs need follow-up fixes** `[Probable — attribution across arXiv 2601.20109 / 2509.14745, exact source unverified]`; 2601.20109 (Tier A preprint): merge success ≠ post-merge quality; issue deltas vanish after churn normalization.
- **The "26% enterprise RCT"**: Cui et al., **Management Science** (DOI 10.1287/mnsc.2025.00535; 4,867 devs, 3 RCTs): +26.08% PRs/week (SE 10.3) — but the tool is **2022-23 autocomplete Copilot**. MUST NOT be cited as evidence for background agents (Tier A with vendor adjacency: 2/6 authors Microsoft Research).
- **METR transfer** (seed; arXiv 2507.09089): per-task randomization + pre-task forecast logging; −19% real vs +24% forecast — perception is anti-signal.
- **METR-lite n=1 protocol (PROPOSAL — synthesis, not a validated instrument, UNVERIFIED)**: coin-flip task assignment before reading the task; log predicted time; track merged/all-started, rework-within-14-days (GitClear churn anchor), corrective follow-ups (2601.16809 anchor), actual-vs-forecast time; ≥20-30 tasks, directional only `[Probable — power heuristic]`.
- **Ecosystem counter-context** (Tier C journalism, 2026 Q1): OSS maintainer overload — curl ended its bug bounty over AI slop; Jazzband shut down; tldraw auto-closes external PRs; GitHub shipped settings to restrict agent PRs. Volume claims ("4M→17M agent PRs in 6 months") = Tier D UNVERIFIED.

## Angle (c) — Effort-scaling & delegation heuristics from published orchestrators

- **Magentic-One** (arXiv 2411.04468, Tier A — refuter-confirmed verbatim): dual ledger. Progress ledger asks 5 questions per round: task complete? / "Is the team looping or repeating itself?" / "Is forward progress being made?" / which agent next? / what instruction. **Stall counter threshold ≤2** → break inner loop → reflection + task-ledger update + replan.
- **SWE-agent** (Tier B, docs + source — refuter-confirmed): budgets cost-denominated; `per_instance_cost_limit` default **3.0 USD**; on per-instance exhaustion → **auto-submit the partial patch** ("exit_cost"). Caveat: the GLOBAL cost limit re-raises WITHOUT auto-submission. Practitioner config: $1 + 50-turn cap (Tier C).
- **Claude Code official docs** (Tier B — refuter-confirmed verbatim quotes): subagents page — delegate when "a side task would flood your main conversation with search results, logs, or file contents you won't reference again"; keep inline when "frequent back-and-forth", "multiple phases share significant context", "quick targeted change", "latency matters". "Subagents cannot spawn other subagents." Agent-teams page — "Start with 3-5 teammates for most workflows"; "5-6 tasks per teammate"; "Three focused teammates often outperform five scattered ones"; teams "use significantly more tokens than a single session"; file-disjoint ownership.
- **OpenHands** (Tier B/C): per-agent `max_iterations` caps **don't compose** under delegation — worst case MAX² global spend (issue #2121 → GLOBAL_MAX_ITERATIONS proposal); enforcement gaps reported (#6857).
- **Aider architect/editor** (Tier B, aider.chat benchmarks): 2-role reasoning/editing split measurably lifted many models vs solo (R1+Sonnet 64.0% polyglot SOTA at ~14× less cost than o1, Jan 2025).
- **Manus** (Tier C, vendor engineering): effort = context/cache economics — "KV-cache hit rate is the single most important metric"; ~50 tool calls/task average; recitation (rewrite todo.md) against lost-in-the-middle; keep failed actions in context; mask tools instead of removing.
- **IFScale** (arXiv 2507.11538, Tier A — refuter-confirmed): instruction-following decays with instruction density — best frontier models **68% accuracy at 500 simultaneous instructions**, bias toward EARLIER instructions. Direct implication for CLAUDE.md/protocol size.
- **Academic gap**: no paper measures the specific "delegation threshold" quality curve (e.g., ≥3 vs ≥4 units); such thresholds are practitioner convention (Tier C), UNVERIFIED as measured optima.

## Counter-evidence

1. **Cognition "Don't Build Multi-Agents"** (Tier D, production stance): share full traces or don't parallelize a writing task; single-threaded + compressor recommended. Contradicts Anthropic's multi-agent post from the same month — for CODING, both converge on inline/shared-context.
2. **MAST** (Tier A, seed): 41-86.7% failure rates; largest bucket = specification/design of the orchestration itself.
3. **Non-monotone scaling** (Tier A, arXiv 2403.02419): more calls degrade hard-query performance; "more agents" is not a safe knob.
4. **IFScale** (Tier A): every always-on protocol rule consumes a decaying adherence budget — more heuristics is itself a measured risk.
5. **Judge quality in production**: Cursor forum reports manual combination beating the judge (Tier C/D); consistent with seed verifier-gap (98.4% coverage → 41.4% selected without mechanical verification).
6. **Best-of-N economics**: Windsurf needed a 50% subsidy; Augment dropped its +3-8pp ensembler as "too expensive" (seed); only measured solo run had 75% wasted attempts (ROI positive only because attempts cost $0.10).

## Verdict

1. **Best-of-N with mechanical selection is unshipped everywhere** — products stop at human/judge selection. For poneglyph it is actionable only as a DIY pattern (`claude -p --worktree` × N + project test suite as selector + human tiebreak), N=2-3, reserved for hard tasks WITH runnable checks. Running it produces novel evidence (gap declared).
2. **Background sessions**: independent data supports a 2-3 session pilot bounded by same-day review capacity — reviewer abandonment is the dominant empirical failure; measure merged/all, rework-14d, corrective follow-ups, actual-vs-forecast (METR-lite proposal; perception is anti-signal).
3. **Heuristics with Tier A support to adopt**: Magentic-One progress-ledger + stall≤2→replan; IFScale-driven protocol brevity (binding rules first). Tier B: cost-budget with partial-salvage, compose caps globally; 3-5 workers / 5-6 tasks / file-disjoint; delegate on output-verbosity, not difficulty.
4. The 26% Management Science RCT is about autocomplete — quarantined from agent claims.
5. Net quality direction of agent code is contested (survives longer, more corrective edits) — design for review throughput, not for raw merge rate.
