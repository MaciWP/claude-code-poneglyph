# US11 — Learning inbox

## Execution prompt (Phase 3 input)

**Task**: Create a Stop hook (`.claude/hooks/learning-inbox.ts`) that, at turn end in sessions with substantive work, appends candidate learnings (user corrections, error→resolution pairs, repeated workarounds) with a confidence score to `.claude/learned/inbox.md`; wire `retro` Phase 5 to consume that inbox as promotion candidates.

**Context**: Pattern proven by everything-claude-code's continuous-learning skill (gated variant). Stop hooks are reliable (error-recovery table). `retro` already produces promotion candidates — this mechanizes capture, not decision. Write tests red→green (tdd forced).

**Constraints**: Capture-only — the hook NEVER writes to skills/rules/memory; only the inbox. Heuristics stay cheap (no LLM calls inside the hook — it's a deterministic script; it can only flag candidates from transcript-visible signals available to hooks). English in files. Registration in settings.json goes last (after US8's edit, single-writer).

**Deliverable**: `learning-inbox.ts` + tests green + `retro/SKILL.md` consumes inbox in Phase 5 + settings.json registration + inbox file format documented in the hook header.

**Verify**: `bun test ./.claude/hooks/` green; simulated Stop payload produces a well-formed inbox entry; retro SKILL.md references the inbox path.

**Ask first**: inbox format (md vs jsonl) if signals are ambiguous — default md (human-readable, retro reads it).

## User story

- **As a**: Oriol
- **I want**: session learnings captured automatically as candidates with my ratification at retro
- **So that**: Commandment IX runs on artefacts, not memory of what happened

## Acceptance criteria

- **AC1**: Given a session end with detectable signals, when the Stop hook runs, then candidates append to `.claude/learned/inbox.md` with type + confidence + source context, and the hook exits silently when no signals (no noise).
- **AC2**: Given retro Phase 5, when reading its skill, then the inbox is a declared input for promotion candidates (consumed + cleared on close).
- **AC3**: Given the hook, when tested, then red→green tests cover entry formatting and the no-signal silent path.
