# US14 — Onboarding de proyectos (suelo de calidad por repo)

## Execution prompt (Phase 3 input)

**Task**: Create a skill (canonical pattern: skill name = command name, no wrapper) — `project-onboard` — that bootstraps the `.claude/` project layer of ANY work repo: (1) analyzes the codebase (stack, test runner, lint/build commands, conventions); (2) generates a LEAN project CLAUDE.md (≤100 lines: commands Claude can't guess, style rules differing from defaults, gotchas — official include/exclude table); (3) writes `.claude/rules/test-policy.md` with the right level (business-critical/mixed/auxiliary) proposed from what the tests cover, user ratifies; (4) wires the discovered test/lint command into the project's verification expectations (so "never report completed without tests" is executable there); (5) optional path-scoped rules when the repo has clear domain boundaries.
**Context**: poneglyph is the global layer; Commandments III/IV/V only bite in real repos if the project layer exists — today it's handmade or absent. Use meta-create patterns + meta-settings-cookbook templates as the build canon. The official CLAUDE.md guidance (≤200 lines, include/exclude) is vendored by US10's rubric work — reference it.
**Constraints**: Generated CLAUDE.md must pass the per-line test ("would removing this cause mistakes?") — generator bias is LEAN, not exhaustive. Everything generated is a PROPOSAL the user ratifies before writing (Commandment I). English templates, runtime Spanish. The skill must work on repos with zero existing .claude/.
**Deliverable**: `project-onboard` skill (SKILL.md + templates/) + registered keywords + a dry-run smoke on a sample repo structure.
**Verify**: smoke: run against a fixture/imaginary repo description → produces CLAUDE.md ≤100 lines + test-policy + verification wiring, all as ratifiable proposals; skill passes the US10 authoring rubric.
**Ask first**: scope of the first version — minimum: CLAUDE.md + test-policy + verification command. Path rules and MCP suggestions can defer.

## User story

- **As a**: Oriol
- **I want**: any repo I work in to get poneglyph's quality floor in one command
- **So that**: the system's value lands on real work, not only on its own meta-repo

## Acceptance criteria

- **AC1**: Given a repo without `.claude/`, when running `/project-onboard`, then it proposes (not writes) a lean CLAUDE.md ≤100 lines + test-policy.md + the discovered verification command, each user-ratified before write.
- **AC2**: Given the generated CLAUDE.md, when audited, then every line passes the official include/exclude guidance (no inferable-from-code content).
- **AC3**: Given the new skill, when audited, then it conforms to the US10 vendored rubric (description, triggers, references).
