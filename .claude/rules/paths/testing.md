## Testing Context

Archivos de tests. Seguir patrones AAA, mantener tests independientes.

- Estructura Arrange-Act-Assert
- Un assert logico por test
- Usar factories/fixtures para datos de prueba
- Mockear dependencias externas, no logica interna
